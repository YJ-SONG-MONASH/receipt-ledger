# 票据账本 使用说明

**票据账本** 是一个离线可用的本地记账网页，用于记录项目支出、个人支出、报销状态、待支付状态和发票附件。

作者信息：**Monash University · Ying-Jian Song**

## 文件说明

项目目录中主要有两个文件：

- `index.html`：记账网页主程序，双击即可打开使用
- `data.json`：本地 JSON 数据备份文件，用于降低浏览器数据丢失风险

## 主要功能

### 1. 新增支出记录

左侧表单可以填写：

- 所属项目
- 支出明细
- 支付人
- 个人/机构
- 金额
- 是否已报销
- 待支付
- 发票信息

点击 **新增记录** 后，数据会保存到浏览器本地。

### 2. 发票附件

发票信息支持上传：

- PDF
- PNG/JPG 等图片

如果记录有附件，表格中“是否开票”显示为 **已开票**。  
如果没有附件，显示为 **未开票**。

### 3. 项目支出统计

页面中有 **项目支出** 区域，会按“所属项目”自动汇总：

- 每个项目总支出
- 项目下有哪些支出
- 项目待支付金额

点击项目卡片里的 **只看**，可以筛选该项目的记录。

### 4. 个人支出统计

页面中有 **个人支出** 区域，会按“支付人”统计个人类型支出。

只有 `个人/机构 = 个人` 的记录会进入个人支出统计。  
机构类型记录不会计入个人支出。

### 5. 导入 PDF/PNG 自动生成记录

点击 **导入PDF/PNG** 可以批量导入票据文件。

文件名格式要求：

```text
机构名-支出明细-金额-日期.pdf
```

示例：

```text
北京大学-会员费用-550-2026.04.13.pdf
清华大学-资料费-1200.50-2026.04.14.png
```

解析规则：

- 第一段：支付人/机构名
- 第二段：支出明细
- 第三段：金额
- 第四段：日期

导入 PDF/PNG 前，必须先在左侧填写或选择 **所属项目**。

如果左侧表单中已经填写了以下字段，会优先使用左侧内容：

- 支出明细
- 支付人
- 金额
- 个人/机构
- 是否已报销
- 待支付

文件名解析结果只作为默认值。

### 6. 导出和导入数据

点击 **导出数据** 可以导出完整 JSON 备份。  
点击 **导入数据** 可以从 JSON 文件恢复账本数据。

导出的 JSON 会包含：

- 支出记录
- 项目信息
- 支付人信息
- 报销/待支付状态
- 发票附件数据

### 7. 绑定 data.json

点击 **绑定data.json**，选择当前项目目录后，网页会自动把账本同步保存到目录下的：

```text
data.json
```

之后新增、删除、导入记录时，会自动更新 `data.json`。

注意：浏览器安全限制要求第一次手动选择目录授权，网页不能自动静默写入本地文件。

### 8. 表格列宽调整

表格支持类似 Excel 的列宽拖动。

把鼠标放到表头列的右边缘，拖动即可调整列宽。  
调整后的列宽会保存在本地，下次打开页面仍然保留。

## 数据保存位置

数据会保存在两个地方：

1. 浏览器 IndexedDB 本地数据库
2. 绑定目录后的 `data.json`

建议使用时点击一次 **绑定data.json**，这样可以降低浏览器清理数据导致账本丢失的风险。

## 推荐使用方式

1. 双击打开 `index.html`
2. 点击 **绑定data.json**
3. 选择项目所在文件夹
4. 开始新增支出或导入 PDF/PNG
5. 定期点击 **导出数据** 做额外备份

## 注意事项

- 不需要联网
- 推荐使用 Chrome 或 Edge
- 不建议频繁清理浏览器站点数据
- 如果换电脑或换浏览器，可以通过 JSON 文件导入恢复数据
- PDF/PNG 自动导入依赖文件名格式，格式不对会跳过导入
